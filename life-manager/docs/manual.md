## はじめに

Life Manager は **GitHub リポジトリをバックエンド**としたタスク管理ツールです。GitHub の Issue・マイルストーン・ラベル機能を活用し、日々のタスク管理、プロジェクト進捗の可視化、定期タスクの自動化などを行います。

データはすべて指定した GitHub リポジトリに保存されるため、別途データベースやサーバーは不要です。

### 必要なもの

- GitHub アカウント
- GitHub Personal Access Token（PAT）
- 管理用リポジトリ（既存でも新規でも可）

## チームメンバーとして参加する場合

リーダーからリポジトリへの招待リンクや「Collaborator に追加した」という連絡を受けた場合の手順です。

### 1. GitHub アカウントを作成する（未登録の場合）

1. ブラウザで **https://github.com** にアクセス
2. 「**Sign up**」をクリック
3. メールアドレス・パスワード・ユーザー名を入力して登録
4. メール認証を完了する

### 2. Collaborator 招待を承認する

招待の受け取り方は 2 パターンあります。

**パターン A: メールで届いた場合**

1. GitHub から届いたメール「You've been invited to collaborate on ...」を開く
2. 「**View invitation**」をクリック
3. GitHub のページが開くので「**Accept invitation**」をクリック

**パターン B: リンクを直接教えてもらった場合**

1. 教えてもらった URL（`https://github.com/オーナー名/リポジトリ名/invitations` など）にアクセス
2. GitHub にログインしていない場合はログイン
3. 「**Accept invitation**」をクリック

<div class="note">
<strong>確認方法:</strong> 招待を承認できたかは、<code>https://github.com/オーナー名/リポジトリ名</code> にアクセスしてリポジトリが表示されれば OK です。「404 Not Found」が出る場合は、招待がまだ届いていないか未承認です。
</div>

### 3. Personal Access Token（PAT）を作成する

招待を承認したら、**自分のアカウントで**トークンを作成します。詳しい手順は次章「初期セットアップ → ステップ 2」を参照してください。

<div class="warn">
<strong>注意:</strong> トークンは必ず自分のアカウントで作成してください。他のメンバーのトークンを共有して使うことはできません（セキュリティ上も推奨されません）。
</div>

### 4. Life Manager をインストール・セットアップする

1. 配布されたインストーラーで Life Manager をインストール
2. 初回起動時のセットアップ画面で以下を入力:

| 項目 | 入力内容 |
|------|----------|
| オーナー名 | **リーダー（リポジトリ所有者）のユーザー名** |
| リポジトリ名 | 共有されたリポジトリ名 |
| トークン | 自分で作成した PAT |

<div class="warn">
<strong>よくある間違い:</strong> オーナー名に自分のユーザー名を入れてしまうケース。必ずリポジトリ所有者の名前を入力してください。リーダーから「<code>alice/project-tasks</code> を使います」と言われたら、オーナー名は <code>alice</code> です。
</div>

3. 「接続してはじめる」をクリックすれば完了です

これでチーム全員が同じリポジトリの Issue を共有し、タスク管理を行えます。

## 初期セットアップ

初回起動時にセットアップ画面が表示されます。3 ステップで完了します。

### ステップ 1: リポジトリの指定

タスクを管理する GitHub リポジトリの **オーナー名**と**リポジトリ名**を入力します。

| 項目 | 入力内容 | 例 |
|------|----------|-----|
| オーナー名 | リポジトリ所有者の GitHub ユーザー名 | `team-leader` |
| リポジトリ名 | リポジトリの名前 | `project-tasks` |

<div class="warn">
<strong>注意:</strong> 自分が Collaborator として参加しているリポジトリの場合も、入力するのは<strong>リポジトリ所有者</strong>のユーザー名です。自分のユーザー名ではありません。
</div>

「次へ」をクリックして進みます。

### ステップ 2: GitHub トークンの作成と入力

GitHub Personal Access Token（PAT）を作成して入力します。

#### PAT の作成手順

1. ブラウザで GitHub にログイン
2. 右上のアイコン →「**Settings**」をクリック
3. 左メニューの一番下「**Developer settings**」をクリック
4. 「**Personal access tokens**」→「**Tokens (classic)**」をクリック
5. 「**Generate new token**」→「**Generate new token (classic)**」をクリック
6. 以下を設定:

| 項目 | 設定内容 |
|------|----------|
| Note | `Life Manager`（わかりやすい名前） |
| Expiration | `No expiration`（推奨）または任意の期限 |
| Select scopes | **`repo`** にチェック（これだけで OK） |

7. 「**Generate token**」をクリック
8. 表示されたトークン（`ghp_` で始まる文字列）をコピー

<div class="warn">
<strong>重要:</strong> トークンは一度しか表示されません。必ずこの画面でコピーしてください。閉じると再表示できません。
</div>

9. Life Manager に戻り、コピーしたトークンを貼り付け
10. 「次へ」をクリック

### ステップ 3: 確認と接続

リポジトリ名とトークンを確認し、「**接続してはじめる**」をクリックします。

接続に成功するとダッシュボード画面に遷移します。

<div class="note">
<strong>ヒント:</strong> 接続後、設定画面の「ラベル一括作成」を実行すると、Life Manager 専用のラベル体系（種別・分野・状態・優先度）が自動でリポジトリに作成されます。チームで利用する場合は最初に一度だけ実行してください。
</div>

## タスク（ダッシュボード）

メインの作業画面です。タスクの確認・作成・管理を行います。

### クイックメモ

画面上部の入力欄からすばやくメモを作成できます。

1. テーマ（分野）をプルダウンから選択
   - 私用 / 仕事 / やりたい / 健康 / 学習
2. テキストを入力
3. Enter キーまたは送信ボタンで作成

作成されたメモは GitHub Issue として登録されます。

### Issue の新規作成

「+ 新規作成」ボタンから詳細な Issue を作成できます。

| 項目 | 説明 |
|------|------|
| タイトル | Issue のタイトル（必須） |
| 本文 | 詳細な説明。マークダウン記法が使えます |
| ラベル | 分類用ラベルを複数選択 |
| マイルストーン | 紐付ける目標を選択 |
| 担当者 | Collaborator から担当者を選択 |

<div class="note">
<strong>タスクリスト:</strong> 「+ タスク項目」ボタンをクリックすると、チェックボックス形式のタスク項目（<code>- [ ] </code>）を本文に挿入できます。
</div>

### フィルタリング

ツールバーのプルダウンで Issue を絞り込めます。

| フィルタ | 内容 |
|----------|------|
| 種別 | イシュー / ルーチン |
| 分野 | 私用 / 仕事 / やりたい / 健康 / 学習 |
| 状態 | 未整理 / 未着手 / 進行中 / チェック待ち / 動作確認 / 完了承認待ち / いつか |
| 優先 | 優先度ラベルによる絞り込み |

「リセット」ボタンで全フィルタを解除できます。

### Issue の操作

各 Issue カードから以下の操作が可能です:

- **クリック** → Issue 詳細画面を開く
- **状態変更** → 進行中・ブロック等のステータスを変更
- **クローズ** → Issue を完了として閉じる

## ボード（カンバン）

Issue をステータスごとの列で視覚的に管理します。

### デフォルトの列

| 列 | 意味 |
|----|------|
| 👤 自分のタスク | 自分が担当者に設定されているタスク（ログイン時に自動表示） |
| 📥 未整理 | まだ分類されていないタスク |
| 📋 未着手 | 着手前のタスク |
| 🔥 進行中 | 現在取り組んでいるタスク |
| 👀 チェック待ち | レビューやチェックを待っているタスク |
| 🧪 動作確認 | 動作確認中のタスク |
| ✅ 完了承認待ち | 完了の承認を待っているタスク |
| 💭 いつか | 将来やりたいタスク |

### 操作方法

- **ドラッグ＆ドロップ** — カードを別の列にドラッグすると、ステータスが自動で変わります
- **「自分のタスク」列へドロップ** — カードを「自分のタスク」列にドラッグすると、自分が担当者として自動設定されます
- **カードクリック** — Issue 詳細画面を開きます
- **フィルタ** — 担当者・優先度・分野・マイルストーンで絞り込めます

### 列のカスタマイズ

ツールバーの ⚙ ボタンから列の設定を変更できます。

- 列の並び順変更（上下移動）
- 列の削除
- 新しい列の追加（カスタムステータス）

## 目標（マイルストーン）

期限付きの目標を管理します。Issue をマイルストーンに紐付けることで、目標に対する進捗を可視化できます。

### マイルストーンの作成

1. 「+ マイルストーン」ボタンをクリック
2. 以下を入力:

| 項目 | 説明 |
|------|------|
| 名前 | マイルストーンのタイトル（必須） |
| 説明 | 詳細な説明（任意） |
| 期日 | 目標の期限日（任意） |

3. 「作成」をクリック

### 進捗の確認

各マイルストーンカードには以下が表示されます:

- **進捗バー** — 完了 Issue の割合を視覚的に表示
- **件数** — クローズ済み / 全 Issue 数
- **期日** — 設定されている場合に表示

## ルーチン

毎日・毎週・毎月、自動的に Issue を作成する機能です。日報や定期チェックなどに活用できます。

### ルーチンの作成

1. 「+ ルーチン追加」ボタンをクリック
2. 以下を設定:

| 項目 | 説明 |
|------|------|
| ルーチン名 | 管理用の名前 |
| 頻度 | 毎日 / 毎週 / 毎月 |
| 曜日 | 毎日の場合、実行する曜日を選択（未選択＝毎日） |
| 実行時刻 | Issue が作成される時刻 |
| Issue タイトル | テンプレート変数が使えます |
| ラベル | 自動付与するラベル |
| 本文 | Issue の本文（任意） |
| 自動クローズ | 指定時刻に自動で Issue を閉じる（任意） |

### テンプレート変数

タイトルや本文で以下の変数が使えます:

| 変数 | 展開例 |
|------|--------|
| `{{date}}` | `2026-03-14` |
| `{{week}}` | `2026年第11週` |
| `{{month}}` | `2026-03` |

**例:** タイトルを `日報 {{date}}` とすると、`日報 2026-03-14` という Issue が自動作成されます。

3. 設定後、「保存」ボタンをクリックして反映

## 日誌（タイムライン）

日ごとの活動記録（ジャーナル）を閲覧・生成する画面です。

### 操作方法

- **日付移動** — 「前日」「翌日」ボタンまたは日付ピッカーで移動
- **「今日」ボタン** — 今日の日付に戻る
- **「手動生成」ボタン** — 選択中の日付のジャーナルを生成
- **「更新」ボタン** — 既存のジャーナルを再取得

ジャーナルにはその日の Issue の活動（作成・クローズ・コメント等）がまとめられます。

## 設定

アプリの各種設定を管理する画面です。

### ラベル管理

- **ラベル一括作成** — Life Manager 用のラベル体系を一括でリポジトリに作成
- **ラベル編集** — 既存ラベルの名前・色・説明を変更
- **ラベル作成** — カスタムラベルの新規作成
- **ラベル削除** — 不要なラベルを削除

### プロジェクト管理

複数のリポジトリを登録して切り替えられます。

- **プロジェクト追加** — 新しいリポジトリを登録（オーナー名/リポジトリ名/表示名/トークン）
- **プロジェクト削除** — 登録済みプロジェクトを削除
- **トークン変更** — プロジェクトごとのトークンを更新

### 通知スケジュール

定期的な通知を設定できます。

| 通知タイプ | 内容 |
|-----------|------|
| 今日のタスク一覧 | 進行中のタスクをリスト表示 |
| 期限超過チェック | 期日を過ぎたタスクを警告 |
| 全体サマリー | オープン/進行中/ブロック件数を表示 |
| カスタムメッセージ | 任意のメッセージを送信 |

通知チャンネル:
- **OS通知** — Windows の通知として表示
- **Discord** — Webhook 経由で Discord チャンネルに送信

### Discord Webhook

Discord チャンネルに通知を送信するための Webhook URL を設定できます。

1. Discord の通知先チャンネルの設定 →「連携サービス」→「ウェブフック」→「新しいウェブフック」
2. Webhook URL をコピー
3. Life Manager の設定画面に貼り付けて保存
4. 「テスト送信」で動作確認

## コマンドパレット

**Ctrl + K** でコマンドパレットを開きます。キーボードだけですばやく操作できます。

| 入力形式 | 動作 | 例 |
|----------|------|-----|
| `m テキスト` | クイックメモを作成 | `m 牛乳を買う` |
| `#番号` | Issue 番号で検索 | `#42` |
| `@ラベル名` | ラベルでフィルタ | `@分野:仕事` |
| `検索語` | タイトル・本文を検索 | `バグ修正` |

Enter キーで実行、Escape キーで閉じます。

## Issue 詳細画面

Issue カードをクリックすると詳細画面（モーダル）が開きます。

### 編集可能な項目

各項目はクリックすると編集モードになります。

| 項目 | 操作 |
|------|------|
| タイトル | クリックして編集、Enter で保存 |
| ラベル | クリックして複数選択、保存ボタンで確定 |
| 担当者 | クリックして選択、保存ボタンで確定 |
| 本文 | クリックして編集、保存ボタンで確定 |

### タスクリスト

本文にチェックボックス（`- [ ]` / `- [x]`）が含まれている場合、進捗バーが表示されます。チェックボックスをクリックして完了/未完了を切り替えられます。

### リマインダー

Issue に対してリマインダーを設定できます。

1. 「+ 設定」ボタンをクリック
2. 通知日時を指定
3. 通知チャンネルを選択（OS通知 / Discord）
4. 「設定」ボタンで登録

設定した日時になると、選択したチャンネルで通知が届きます。

### コメント

Issue に対してコメントを追加できます。

1. テキストエリアにコメントを入力
2. 「コメント追加」ボタンまたは **Ctrl + Enter** で投稿

コメントは GitHub 上の Issue コメントとして保存されます。

### クローズ / リオープン

- **クローズ** — タスクを完了として閉じる
- **リオープン** — 閉じたタスクを再度開く

## ラベル体系

Life Manager は以下のラベル体系でタスクを分類します。初期セットアップ後、設定画面の「ラベル一括作成」で自動作成できます。

### 種別（タスクの種類）

| ラベル | 用途 |
|--------|------|
| 種別:イシュー | 通常のタスクや課題 |
| 種別:ルーチン | 定期的に自動作成されるタスク |

### 分野（カテゴリ）

| ラベル | 用途 |
|--------|------|
| 分野:私用 | プライベートのタスク |
| 分野:仕事 | 仕事関連のタスク |
| 分野:やりたい | やりたいこと・願望 |
| 分野:健康 | 健康・運動関連 |
| 分野:学習 | 勉強・スキルアップ関連 |

### 状態（進捗ステータス）

| ラベル | 用途 |
|--------|------|
| 状態:未整理 | まだ整理されていない |
| 状態:未着手 | 整理済みだが未着手 |
| 状態:進行中 | 現在取り組んでいる |
| 状態:チェック待ち | レビューやチェックを待っている |
| 状態:動作確認 | 動作確認中 |
| 状態:完了承認待ち | 完了の承認を待っている |
| 状態:いつか | 将来やりたい |

### 優先度

優先度ラベルはカスタムで作成できます。設定画面のラベル管理から追加してください。

## よくある質問

### Q: データはどこに保存されますか？

A: すべて GitHub リポジトリの Issue・マイルストーン・ラベルとして保存されます。アプリをアンインストールしてもデータは GitHub 上に残ります。ルーチンや通知の設定はリポジトリの `config/` フォルダに YAML ファイルとして保存されます。

### Q: トークンの期限が切れたら？

A: 設定画面からトークンを更新できます。GitHub で新しい PAT を作成し、設定画面の「トークン変更」から入力してください。

### Q: 複数のリポジトリを使えますか？

A: はい。設定画面の「プロジェクト管理」から複数のリポジトリを登録し、ヘッダーのプルダウンで切り替えられます。プロジェクトごとに異なるトークンを設定することも可能です。

### Q: チームで使うには？

A: リポジトリの所有者がメンバーを Collaborator として招待します。各メンバーは自分の GitHub アカウントで PAT を作成し、Life Manager に同じリポジトリを設定します。

### Q: オフラインで使えますか？

A: GitHub API への接続が必要なため、オフラインでは使用できません。
