## はじめに

Life Manager は **GitHub リポジトリをバックエンド**としたタスク管理ツールです。GitHub の Issue・マイルストーン・ラベル機能を活用し、日々のタスク管理、プロジェクト進捗の可視化、定期タスクの自動化などを行います。

データはすべて指定した GitHub リポジトリに保存されるため、別途データベースやサーバーは不要です。

### 必要なもの

- GitHub アカウント
- GitHub Personal Access Token（PAT）
- 管理用リポジトリ（既存でも新規でも可）

## チームメンバーとして参加する場合

リーダーからリポジトリへの招待リンクや「Collaborator に追加した」という連絡を受けた場合の手順です。

### 1. GitHub アカウントを作成する（未登録の場合）

1. ブラウザで **https://github.com** にアクセス
2. 「**Sign up**」をクリック
3. メールアドレス・パスワード・ユーザー名を入力して登録
4. メール認証を完了する

### 2. Collaborator 招待を承認する

招待の受け取り方は 2 パターンあります。

**パターン A: メールで届いた場合**

1. GitHub から届いたメール「You've been invited to collaborate on ...」を開く
2. 「**View invitation**」をクリック
3. GitHub のページが開くので「**Accept invitation**」をクリック

**パターン B: リンクを直接教えてもらった場合**

1. 教えてもらった URL（`https://github.com/オーナー名/リポジトリ名/invitations` など）にアクセス
2. GitHub にログインしていない場合はログイン
3. 「**Accept invitation**」をクリック

<div class="note">
<strong>確認方法:</strong> 招待を承認できたかは、<code>https://github.com/オーナー名/リポジトリ名</code> にアクセスしてリポジトリが表示されれば OK です。「404 Not Found」が出る場合は、招待がまだ届いていないか未承認です。
</div>

### 3. Personal Access Token（PAT）を作成する

招待を承認したら、**自分のアカウントで**トークンを作成します。詳しい手順は次章「初期セットアップ → ステップ 2」を参照してください。

<div class="warn">
<strong>注意:</strong> トークンは必ず自分のアカウントで作成してください。他のメンバーのトークンを共有して使うことはできません（セキュリティ上も推奨されません）。
</div>

### 4. Life Manager をインストール・セットアップする

1. 配布されたインストーラーで Life Manager をインストール
2. 初回起動時のセットアップ画面で以下を入力:

| 項目 | 入力内容 |
|------|----------|
| オーナー名 | **リーダー（リポジトリ所有者）のユーザー名** |
| リポジトリ名 | 共有されたリポジトリ名 |
| トークン | 自分で作成した PAT |

<div class="warn">
<strong>よくある間違い:</strong> オーナー名に自分のユーザー名を入れてしまうケース。必ずリポジトリ所有者の名前を入力してください。リーダーから「<code>alice/project-tasks</code> を使います」と言われたら、オーナー名は <code>alice</code> です。
</div>

3. 「接続してはじめる」をクリックすれば完了です

これでチーム全員が同じリポジトリの Issue を共有し、タスク管理を行えます。

## チーム運用（管理者向け）

リポジトリ所有者（リーダー）がチーム利用のためにリポジトリを準備する手順です。

### Organization の利用を推奨

2 人以上で継続的に作業する場合は、**GitHub Organization**（無料）の利用を推奨します。

| | 個人リポジトリ | Organization |
|---|---|---|
| 権限管理 | Collaborator 単位 | Team 単位で一括管理 |
| リポジトリ所有 | 個人に紐づく | Org に紐づく（人が抜けても残る） |
| Fine-grained PAT | オーナーのみリポ個別指定可能 | **メンバー全員がリポ個別指定可能** |

### 既存リポジトリを Organization に移行する

すでに個人アカウントで運用しているリポジトリを Organization に移すことができます。

<div class="step">

1. Organization を作成（未作成の場合: Settings → Organizations → New organization、Free プランで可）
2. リポジトリの **Settings** → **General** → 最下部「**Danger Zone**」→「**Transfer ownership**」
3. 移行先に Organization 名を入力し、確認して転送
4. リポジトリの URL が `個人名/リポ名` → `org名/リポ名` に変わる
5. 各メンバーの Life Manager でプロジェクト設定のオーナー名を Organization 名に更新
6. 各メンバーが Fine-grained PAT を再発行（Resource owner を Organization に変更し、対象リポジトリを選択）

</div>

<div class="warn">
<strong>注意:</strong> 転送後、旧 URL からのリダイレクトは一定期間有効ですが、GitHub API のエンドポイントは新 URL（<code>org名/リポ名</code>）を使用する必要があります。Issue 番号やデータはすべて維持されます。
</div>

### Organization を使う場合（推奨）

**リポジトリ所有者（リーダー）が行うこと:**

1. GitHub Organization を作成（Settings → Organizations → New organization、Free プランで可）
2. Organization 内にリポジトリを作成（Private 推奨）
3. メンバーを Organization に招待（Organization Settings → Members → Invite member）
4. メンバーに **Write** 以上のロールを付与（Issue 操作・Contents 書き込みに必要）

**各メンバーが行うこと:**

1. Organization 招待を承認
2. GitHub → Settings → Developer settings → Personal access tokens → Fine-grained tokens で PAT を発行
3. トークン設定:
   - **Resource owner**: Organization 名を選択
   - **Repository access**: 「Only select repositories」→ 対象リポジトリを選択
   - **Permissions**: 下記「必要な権限」を参照
4. Life Manager にトークンとリポジトリ（`org名/リポ名`）を設定

### 個人リポジトリの場合

**リポジトリ所有者（リーダー）が行うこと:**

1. リポジトリを作成（Private 推奨）
2. メンバーを Collaborator として招待（Settings → Collaborators → Add people）
3. メンバーに **Write** 以上のロールを付与

**各メンバーが行うこと:**

1. Collaborator 招待を承認
2. トークンを発行（下記「Fine-grained PAT の注意点」を参照）
3. Life Manager にトークンとリポジトリを設定

### 必要な権限（Fine-grained PAT）

Fine-grained PAT を使用する場合、以下の権限を設定してください。

| Permission | Access | 用途 |
|-----------|--------|------|
| **Issues** | Read and Write | Issue・コメントの作成/編集/クローズ |
| **Contents** | Read and Write | ルーチン・通知設定・ボード設定の保存 |
| **Metadata** | Read | 自動付与（リポジトリ情報の取得） |

### Fine-grained PAT の注意点

<div class="warn">
<strong>制限事項:</strong> Fine-grained PAT でリポジトリを個別指定する場合、選択肢に表示されるのは<strong>自分がオーナーのリポジトリ</strong>と<strong>所属 Organization 内のリポジトリ</strong>のみです。他人の個人リポジトリに Collaborator として招待されたものは表示されません。
</div>

**対処法:**

| 方法 | 説明 |
|------|------|
| Organization に移行 | メンバー全員が Fine-grained でリポ個別指定可能になる |
| Classic PAT を使用 | `repo` スコープで招待先リポジトリにもアクセス可能 |
| Fine-grained「All repositories」 | 全リポジトリ対象（スコープが広い） |

Life Manager ではプロジェクトごとにトークンを設定できるため、自分のリポには Fine-grained、招待リポには Classic と使い分けが可能です。

### トークンの解決順序

アプリ起動時・プロジェクト切替時のトークン解決:

1. **プロジェクト固有トークン** — プロジェクト設定でトークンが登録されていればそれを使用
2. **グローバルトークン** — 初期セットアップで登録したトークンにフォールバック

## 初期セットアップ

初回起動時にセットアップ画面が表示されます。3 ステップで完了します。

### ステップ 1: リポジトリの指定

タスクを管理する GitHub リポジトリの **オーナー名**と**リポジトリ名**を入力します。

| 項目 | 入力内容 | 例 |
|------|----------|-----|
| オーナー名 | リポジトリ所有者の GitHub ユーザー名 | `team-leader` |
| リポジトリ名 | リポジトリの名前 | `project-tasks` |

<div class="warn">
<strong>注意:</strong> 自分が Collaborator として参加しているリポジトリの場合も、入力するのは<strong>リポジトリ所有者</strong>のユーザー名です。自分のユーザー名ではありません。
</div>

「次へ」をクリックして進みます。

### ステップ 2: GitHub トークンの作成と入力

GitHub Personal Access Token（PAT）を作成して入力します。

#### PAT の作成手順

トークンには **Fine-grained token（推奨）** と **Classic token** の 2 種類があります。

**方法 A: Fine-grained token（推奨）**

1. ブラウザで GitHub にログイン
2. 右上のアイコン →「**Settings**」→「**Developer settings**」
3. 「**Personal access tokens**」→「**Fine-grained tokens**」→「**Generate new token**」
4. 以下を設定:

| 項目 | 設定内容 |
|------|----------|
| Token name | `Life Manager`（わかりやすい名前） |
| Expiration | 任意の期限 |
| Resource owner | 自分のアカウントまたは Organization |
| Repository access | 「Only select repositories」→ 対象リポジトリを選択 |

5. **Permissions** で以下を設定:

| Permission | Access |
|-----------|--------|
| **Issues** | Read and Write |
| **Contents** | Read and Write |
| **Metadata** | Read（自動付与） |

6. 「**Generate token**」をクリック
7. 表示されたトークン（`github_pat_` で始まる文字列）をコピー

**方法 B: Classic token**

1. 「**Personal access tokens**」→「**Tokens (classic)**」→「**Generate new token (classic)**」
2. 以下を設定:

| 項目 | 設定内容 |
|------|----------|
| Note | `Life Manager` |
| Expiration | `No expiration`（推奨）または任意の期限 |
| Select scopes | **`repo`** にチェック |

3. 「**Generate token**」をクリック
4. 表示されたトークン（`ghp_` で始まる文字列）をコピー

<div class="warn">
<strong>重要:</strong> トークンは一度しか表示されません。必ずこの画面でコピーしてください。閉じると再表示できません。
</div>

<div class="note">
<strong>どちらを選ぶ？:</strong> Organization のリポジトリを使う場合や、アクセス範囲を最小限にしたい場合は Fine-grained token を推奨します。他人の個人リポジトリに Collaborator として招待された場合は Classic token を使ってください（詳しくは「Fine-grained PAT の注意点」を参照）。
</div>

8. Life Manager に戻り、コピーしたトークンを貼り付け
9. 「次へ」をクリック

### ステップ 3: 確認と接続

リポジトリ名とトークンを確認し、「**接続してはじめる**」をクリックします。

接続に成功するとダッシュボード画面に遷移します。

<div class="note">
<strong>ヒント:</strong> 接続後、設定画面の「ラベル一括作成」を実行すると、Life Manager 専用のラベル体系（種別・分野・状態・優先度）が自動でリポジトリに作成されます。チームで利用する場合は最初に一度だけ実行してください。
</div>

## 画面構成

アプリは上部のナビゲーションバーで画面を切り替えます。

| アイコン | 画面名 | 説明 |
|----------|--------|------|
| 📋 | タスク | メイン画面。タスクの一覧・作成・フィルタリング |
| 📊 | ボード | カンバン形式でタスクの状態を可視化 |
| 🎯 | 目標 | マイルストーン（期限付きの目標）の管理 |
| 📊 | ガント | マイルストーン内の Issue をガントチャートで可視化 |
| 🔄 | ルーチン | 定期的に自動作成されるタスクの設定 |
| 📅 | 日誌 | 日ごとの活動ジャーナルの閲覧・生成 |
| ⚙️ | 設定 | ラベル・通知・プロジェクト等の設定 |

ヘッダー左上のプルダウンメニューで、複数のプロジェクト（リポジトリ）を切り替えられます。

## タスク（ダッシュボード）

メインの作業画面です。タスクの確認・作成・管理を行います。

### クイックメモ

画面上部の入力欄からすばやくメモを作成できます。

1. テーマ（分野）をプルダウンから選択
   - 私用 / 仕事 / やりたい / 健康 / 学習
2. テキストを入力
3. Enter キーまたは送信ボタンで作成

作成されたメモは GitHub Issue として登録されます。

### Issue の新規作成

「+ 新規作成」ボタンから詳細な Issue を作成できます。

| 項目 | 説明 |
|------|------|
| タイトル | Issue のタイトル（必須） |
| 本文 | 詳細な説明。マークダウン記法が使えます |
| ラベル | 分類用ラベルを複数選択 |
| マイルストーン | 紐付ける目標を選択 |
| 担当者 | Collaborator から担当者を選択 |

<div class="note">
<strong>タスクリスト:</strong> 「+ タスク項目」ボタンをクリックすると、チェックボックス形式のタスク項目（<code>- [ ] </code>）を本文に挿入できます。
</div>

### フィルタリング

ツールバーのプルダウンで Issue を絞り込めます。

| フィルタ | 内容 |
|----------|------|
| 種別 | イシュー / ルーチン |
| 分野 | 私用 / 仕事 / やりたい / 健康 / 学習 |
| 状態 | 未整理 / 未着手 / 進行中 / チェック待ち / 動作確認 / 完了承認待ち / いつか |
| 優先 | 優先度ラベルによる絞り込み |

「リセット」ボタンで全フィルタを解除できます。

### Issue の操作

各 Issue カードから以下の操作が可能です:

- **クリック** → Issue 詳細画面を開く
- **状態変更** → 進行中・ブロック等のステータスを変更
- **クローズ** → Issue を完了として閉じる

## ボード（カンバン）

Issue をステータスごとの列で視覚的に管理します。

### デフォルトの列

| 列 | 意味 |
|----|------|
| 👤 自分のタスク | 自分が担当者に設定されているタスク（ログイン時に自動表示） |
| 📥 未整理 | まだ分類されていないタスク |
| 📋 未着手 | 着手前のタスク |
| 🔥 進行中 | 現在取り組んでいるタスク |
| 👀 チェック待ち | レビューやチェックを待っているタスク |
| 🧪 動作確認 | 動作確認中のタスク |
| ✅ 完了承認待ち | 完了の承認を待っているタスク |
| 💭 いつか | 将来やりたいタスク |

### 操作方法

- **ドラッグ＆ドロップ** — カードを別の列にドラッグすると、ステータスが自動で変わります
- **「自分のタスク」列へドロップ** — カードを「自分のタスク」列にドラッグすると、自分が担当者として自動設定されます
- **カードクリック** — Issue 詳細画面を開きます
- **フィルタ** — 担当者・優先度・分野・マイルストーンで絞り込めます

### 列のカスタマイズ

ツールバーの ⚙ ボタンから列の設定を変更できます。

- 列の並び順変更（上下移動）
- 列の削除
- 新しい列の追加（カスタムステータス）

## 目標（マイルストーン）

期限付きの目標を管理します。Issue をマイルストーンに紐付けることで、目標に対する進捗を可視化できます。

### マイルストーンの作成

1. 「+ マイルストーン」ボタンをクリック
2. 以下を入力:

| 項目 | 説明 |
|------|------|
| 名前 | マイルストーンのタイトル（必須） |
| 説明 | 詳細な説明（任意） |
| 期日 | 目標の期限日（任意） |

3. 「作成」をクリック

### 進捗の確認

各マイルストーンカードには以下が表示されます:

- **進捗バー** — 完了 Issue の割合を視覚的に表示
- **件数** — クローズ済み / 全 Issue 数
- **期日** — 設定されている場合に表示

## ガントチャート

マイルストーン内の Issue をタイムラインで可視化する画面です。Canvas 描画による高速レンダリングに対応しています。

### 基本操作

1. 画面上部のプルダウンからマイルストーンを選択（前回選択したマイルストーンは自動で記憶されます）
2. Issue にガントメタデータ（開始日・終了日）が設定されていると、タイムライン上にバーとして表示されます

### フィルタリング

| フィルタ | 内容 |
|----------|------|
| 担当者 | 特定の担当者の Issue のみ表示 |
| 状態 | 進行中・未着手などのステータスで絞り込み |
| 分野 | 仕事・私用などのカテゴリで絞り込み |

### タイムスケール

ツールバーから表示単位を切り替えられます。

| スケール | 表示 |
|----------|------|
| 日 | 1 日ごとのグリッド |
| 週 | 1 週間ごとのグリッド |
| 月 | 1 ヶ月ごとのグリッド |

### スクロール操作

- **横スクロールバー**（画面上部）をドラッグしてタイムラインを横移動
- **Canvas 上でマウスドラッグ**で縦横自由にスクロール
- 「今日」ボタンで今日の日付にジャンプ

### ガントメタデータの設定

Issue 詳細画面の「ガントチャート」セクションから設定します。データは Issue 本文に HTML コメントとして保存されるため、GitHub 上でも見た目に影響しません。

| 項目 | 説明 |
|------|------|
| 開始日 / 終了日 | タイムライン上のバーの位置と長さ |
| 依存関係 | 先行タスクの Issue 番号（矢印で接続表示） |
| 進捗モード | checkbox（タスクリスト）/ manual（手動入力）/ binary（完了/未完了） |
| 進捗値 | manual モード時の進捗パーセント |

<div class="note">
<strong>ヒント:</strong> 依存関係を設定すると、先行タスクから後続タスクへ矢印が描画されます。タスク間の順序関係を視覚的に把握できます。
</div>

**Issue 本文に保存されるメタデータ形式（参考）:**

```
<!-- gantt:2026-03-10/2026-03-20 -->    開始日/終了日
<!-- depends:#12,#15 -->                依存関係（先行タスク）
<!-- progress-mode:manual -->           進捗モード
<!-- progress:60 -->                    手動進捗値
```

## ルーチン

毎日・毎週・毎月、自動的に Issue を作成する機能です。日報や定期チェックなどに活用できます。

### ルーチンの作成

1. 「+ ルーチン追加」ボタンをクリック
2. 以下を設定:

| 項目 | 説明 |
|------|------|
| ルーチン名 | 管理用の名前 |
| 頻度 | 毎日 / 毎週 / 毎月 |
| 曜日 | 毎日の場合、実行する曜日を選択（未選択＝毎日） |
| 実行時刻 | Issue が作成される時刻 |
| Issue タイトル | テンプレート変数が使えます |
| ラベル | 自動付与するラベル |
| 本文 | Issue の本文（任意） |
| 自動クローズ | 指定時刻に自動で Issue を閉じる（任意） |

### テンプレート変数

タイトルや本文で以下の変数が使えます:

| 変数 | 展開例 |
|------|--------|
| `{{date}}` | `2026-03-14` |
| `{{week}}` | `2026年第11週` |
| `{{month}}` | `2026-03` |

**例:** タイトルを `日報 {{date}}` とすると、`日報 2026-03-14` という Issue が自動作成されます。

3. 設定後、「保存」ボタンをクリックして反映

## 設定

アプリの各種設定を管理する画面です。

### ラベル管理

- **ラベル一括作成** — Life Manager 用のラベル体系を一括でリポジトリに作成
- **ラベル編集** — 既存ラベルの名前・色・説明を変更
- **ラベル作成** — カスタムラベルの新規作成
- **ラベル削除** — 不要なラベルを削除

### プロジェクト管理

複数のリポジトリを登録して切り替えられます。

- **プロジェクト追加** — 新しいリポジトリを登録（オーナー名/リポジトリ名/表示名/トークン）
- **プロジェクト削除** — 登録済みプロジェクトを削除
- **トークン変更** — プロジェクトごとのトークンを更新

### 通知スケジュール

定期的な通知を設定できます。

| 通知タイプ | 内容 |
|-----------|------|
| 今日のタスク一覧 | 進行中のタスクをリスト表示 |
| 期限超過チェック | 期日を過ぎたタスクを警告 |
| 全体サマリー | オープン/進行中/ブロック件数を表示 |
| カスタムメッセージ | 任意のメッセージを送信 |

通知チャンネル:
- **OS通知** — Windows の通知として表示
- **Discord** — Webhook 経由で Discord チャンネルに送信

### マニュアル

設定画面の「マニュアルを開く」ボタンから、このマニュアル（PDF）をいつでも参照できます。

### Discord Webhook

Discord チャンネルに通知を送信するための Webhook URL を設定できます。

1. Discord の通知先チャンネルの設定 →「連携サービス」→「ウェブフック」→「新しいウェブフック」
2. Webhook URL をコピー
3. Life Manager の設定画面に貼り付けて保存
4. 「テスト送信」で動作確認

<div class="warn">
<strong>スマホで通知が届かない場合:</strong> Android では Discord アプリがバッテリー最適化の対象になっていると、バックグラウンドでの通知が遅延・抑制されることがあります。以下を確認してください。
<ul>
<li><strong>バッテリー最適化を無効にする</strong> — 設定 → アプリ → Discord → バッテリー →「制限なし」に変更</li>
<li><strong>他デバイスのセッションを整理する</strong> — PC 版 Discord を起動したままだと、スマホへの通知が抑制されます。PC で Discord を使い終わったら終了してください</li>
<li><strong>チャンネルの通知設定</strong> — Discord アプリで通知先チャンネルの通知設定が「すべてのメッセージ」になっていることを確認</li>
</ul>
</div>

## Issue 詳細画面

Issue カードをクリックすると詳細画面（モーダル）が開きます。

### 編集可能な項目

各項目はクリックすると編集モードになります。

| 項目 | 操作 |
|------|------|

本文にチェックボックス（`- [ ]` / `- [x]`）が含まれている場合、進捗バーが表示されます。チェックボックスをクリックして完了/未完了を切り替えられます。

1. テキストエリアにコメントを入力
2. 「コメント追加」ボタンまたは **Ctrl + Enter** で投稿

コメントは GitHub 上の Issue コメントとして保存されます。

### クローズ / リオープン

## ラベル体系

Life Manager は以下のラベル体系でタスクを分類します。初期セットアップ後、設定画面の「ラベル一括作成」で自動作成できます。

### 種別（タスクの種類）

| ラベル | 用途 |
|--------|------|
| 種別:イシュー | 通常のタスクや課題 |
| 種別:ルーチン | 定期的に自動作成されるタスク |

### 分野（カテゴリ）

| ラベル | 用途 |
|--------|------|
| 分野:私用 | プライベートのタスク |
| 分野:仕事 | 仕事関連のタスク |
| 分野:やりたい | やりたいこと・願望 |
| 分野:健康 | 健康・運動関連 |
| 分野:学習 | 勉強・スキルアップ関連 |

### 状態（進捗ステータス）

| ラベル | 用途 |
|--------|------|
| 状態:未整理 | まだ整理されていない |
| 状態:未着手 | 整理済みだが未着手 |
| 状態:進行中 | 現在取り組んでいる |
| 状態:チェック待ち | レビューやチェックを待っている |
| 状態:動作確認 | 動作確認中 |
| 状態:完了承認待ち | 完了の承認を待っている |
| 状態:いつか | 将来やりたい |

### 優先度

優先度ラベルはカスタムで作成できます。設定画面のラベル管理から追加してください。

## よくある質問

### Q: データはどこに保存されますか？

A: すべて GitHub リポジトリの Issue・マイルストーン・ラベルとして保存されます。アプリをアンインストールしてもデータは GitHub 上に残ります。ルーチンや通知の設定はリポジトリの `config/` フォルダに YAML ファイルとして保存されます。

### Q: トークンの期限が切れたら？

A: 設定画面からトークンを更新できます。GitHub で新しい PAT を作成し、設定画面の「トークン変更」から入力してください。

### Q: 複数のリポジトリを使えますか？

A: はい。設定画面の「プロジェクト管理」から複数のリポジトリを登録し、ヘッダーのプルダウンで切り替えられます。プロジェクトごとに異なるトークンを設定することも可能です。

### Q: チームで使うには？

A: 2 人以上で使う場合は **GitHub Organization**（無料）の利用を推奨します。Organization 内にリポジトリを作成し、メンバーを招待してください。各メンバーは自分のアカウントで Fine-grained PAT を発行し、Life Manager に同じリポジトリを設定します。詳しくは「チーム運用（管理者向け）」を参照してください。

### Q: Fine-grained PAT でリポジトリが選択肢に出ません

A: Fine-grained PAT のリポジトリ個別指定では、自分がオーナーのリポジトリと所属 Organization のリポジトリのみが表示されます。他人の個人リポジトリに Collaborator として招待された場合は表示されません。対処法は「Fine-grained PAT の注意点」を参照してください。

### Q: ガントチャートにバーが表示されません

A: Issue 詳細画面の「ガントチャート」セクションから開始日・終了日を設定してください。メタデータが未設定の Issue はタイムライン上に表示されません。

### Q: オフラインで使えますか？

A: GitHub API への接続が必要なため、オフラインでは使用できません。
